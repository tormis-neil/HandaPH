1. Introduction

Purpose of the System

The HandaPH system is a personalized typhoon preparedness web application. It aims to assist Filipino families in building comprehensive Go-Bags and actionable emergency plans based on their specific needs, location, and household size.

Overview

By automating preparedness checklists and debunking dangerous typhoon myths, HandaPH bridges the gap between general advisories and actionable, personalized safety measures, aiming to reduce last-minute panic and save lives.

Objectives

  • Provide customized typhoon survival checklists.
  • Educate the public on essential Go-Bag items and budget alternatives.
  • Debunk prevalent typhoon myths with verified facts.
  • Allow administrators to manage data, feedback, and view submission analytics.

2. System Architecture Overview

HandaPH is built on the Laravel (PHP 8.2+) framework, utilizing a robust Model-View-Controller (MVC) architecture. It uses Blade templating for the frontend views, Eloquent ORM for database interactions, and MySQL for data persistence.

  • Frontend: Blade Templates, HTML5, Vanilla CSS3, JavaScript.
  • Backend: Laravel Framework (PHP), Controllers, Middleware, Form Requests.
  • Database: MySQL relational database.

5.1 Authentication Logic

File Path: app/Http/Controllers/Auth/AuthenticatedSessionController.php

Manages login and logout processing.

<?php
namespace App\Http\Controllers\Auth;

use App\Http\Controllers\Controller;
use App\Http\Requests\Auth\LoginRequest;
use Illuminate\Http\RedirectResponse;
use Illuminate\Support\Facades\Auth;

class AuthenticatedSessionController extends Controller
{
    public function store(LoginRequest $request): RedirectResponse
    {
        $request->authenticate();
        $request->session()->regenerate();

        return redirect()->intended(route('admin.dashboard', absolute: false));
    }

    public function destroy(Request $request): RedirectResponse
    {
        Auth::guard('web')->logout();
        $request->session()->invalidate();
        $request->session()->regenerateToken();

        return redirect('/');
    }
}

5.3 Business Logic: Checklist Rule Engine

File Path: app/Services/ChecklistRuleEngine.php

Filters rules based on the user's survey submission parameters using Laravel Collections.

<?php
namespace App\Services;

use App\Models\ChecklistRule;

class ChecklistRuleEngine
{
    public function filter(array $selections): array
    {
        $rules = ChecklistRule::where('is_active', true)->get();

        $matched = $rules->filter(function ($rule) use ($selections) {
            return $this->matches($rule, $selections);
        });

        return [
            'before' => $matched->where('phase', 'before')->values(),
            'during' => $matched->where('phase', 'during')->values(),
            'after'  => $matched->where('phase', 'after')->values(),
        ];
    }

    private function matches(ChecklistRule $rule, array $selections): bool
    {
        $locMatch  = empty($rule->locations) || in_array($selections['location'], $rule->locations, true);
        $sizeMatch = empty($rule->sizes) || in_array($selections['household_size'], $rule->sizes, true);
        $needsMatch = empty($rule->special_needs) || count(array_intersect($rule->special_needs, $selections['special_needs'])) > 0;
        $houseMatch = empty($rule->house_types) || in_array($selections['house_type'], $rule->house_types, true);

        return $locMatch && $sizeMatch && $needsMatch && $houseMatch;
    }
}

6. Database Structure

The system relies on a relational database design managed through Laravel migrations.

6.1 Users Table

Stores administrator credentials securely using hashed passwords.

Schema::create('users', function (Blueprint $table) {
    $table->id();
    $table->string('name');
    $table->string('email')->unique();
    $table->timestamp('email_verified_at')->nullable();
    $table->string('password'); // e.g., DB_PASSWORD=********
    $table->rememberToken();
    $table->timestamps();
});

6.2 Go-Bag Items Table

Stores recommended emergency items.

Schema::create('go_bag_items', function (Blueprint $table) {
    $table->id();
    $table->enum('category', ['essentials', 'recommended', 'optional']);
    $table->string('name');
    $table->text('description')->nullable();
    $table->text('budget_alternative')->nullable();
    $table->boolean('is_active')->default(true);
    $table->timestamps();
});

6.3 Checklist Rules Table

Stores logic for checklist generation. Uses JSON columns for flexibility in matching parameters.

Schema::create('checklist_rules', function (Blueprint $table) {
    $table->id();
    $table->text('item_text');
    $table->enum('phase', ['before', 'during', 'after']);
    $table->string('tag');
    $table->string('tag_class');
    $table->json('locations');
    $table->json('sizes');
    $table->json('special_needs');
    $table->json('house_types');
    $table->boolean('is_active')->default(true);
    $table->timestamps();
});

6.4 Survey Submissions Table

Stores anonymous responses to generate analytics.

Schema::create('survey_submissions', function (Blueprint $table) {
    $table->id();
    $table->enum('location', ['coastal', 'mountainous', 'inland', 'flood-prone']);
    $table->enum('household_size', ['1', '2-4', '5-7', '8-plus']);
    $table->json('special_needs');
    $table->enum('house_type', ['light', 'semi-concrete', 'concrete']);
    $table->timestamps();
});

9. Analytics and Feedback Module

The platform collects anonymous survey data and feedback to help authorities understand demographics.

9.1 Analytics Controller

File Path: app/Http/Controllers/Admin/AnalyticsController.php

<?php
namespace App\Http\Controllers\Admin;

use App\Http\Controllers\Controller;
use App\Models\SurveySubmission;
use Illuminate\Http\JsonResponse;

class AnalyticsController extends Controller
{
    public function summary(): JsonResponse
    {
        return response()->json([
            'location' => $this->locationDistribution(),
            'household_size' => $this->householdSizeDistribution(),
            'special_needs' => $this->specialNeedsDistribution(),
            'house_type' => $this->houseTypeDistribution(),
        ]);
    }

    private function locationDistribution(): array
    {
        $rows = SurveySubmission::selectRaw('location, COUNT(*) as count')
            ->groupBy('location')
            ->pluck('count', 'location')
            ->toArray();

        return [
            'coastal'     => $rows['coastal']     ?? 0,
            'mountainous' => $rows['mountainous'] ?? 0,
            'inland'      => $rows['inland']      ?? 0,
            'flood-prone' => $rows['flood-prone'] ?? 0,
        ];
    }
}

10. Security Features

  • Cross-Site Request Forgery (CSRF) Protection: All state-changing forms include Laravel's @csrf directive.
  • Password Hashing: User passwords are encrypted using bcrypt (Hash::make).
  • Environment Configuration: Sensitive variables are kept out of source code using the .env file. Ex: DB_PASSWORD=********, APP_KEY=********.
  • Input Validation: Form Requests are used to validate and sanitize all user input before processing.
  • Rate Limiting: Endpoints like login and feedback submission use throttle middleware to prevent brute-force or spam.
